Le Principe YAGNI

Définition — Ron Jeffries, XP Explained (1999)

"Always implement things when you actually need them, never when you just foresee that you need them."

YAGNI s'oppose à la tentation du développeur de prévoir des fonctionnalités "au cas où". Chaque ligne de code non requise aujourd'hui est une dette : elle doit être testée, maintenue, documentée, et elle complexifie inutilement le système.

Code mort typique (violations YAGNI)

  • • Champs non utilisés dans les entités (isPaid, vipTier…)
  • • Interfaces sans implémentation (INotificationService)
  • • Méthodes avec // implement later
  • • Services entiers connectés à rien (WeatherService)
  • • Classes lancant UnsupportedOperationException

Après nettoyage YAGNI

  • • Seuls les champs du besoin actuel validé existent
  • • Zéro interface non implémentée dans le projet
  • • Zéro méthode vide ou TODO
  • • SonarQube : 0 "Dead Code" warning
  • • Le code reflète exactement ce que le produit fait maintenant

Mission

Besoin réel validé par le sponsor

  • POST /api/reservations — créer une réservation
  • GET /api/reservations — lister toutes les réservations
  • Chaque réservation contient : nom du groupe, date/heure d'arrivée, nombre de personnes (1 à 10)
  • Un nombre de personnes hors de [1, 10] doit retourner une erreur 400
  • Stockage en mémoire (liste statique) — pas de base de données requise

Tâches

  1. 1

    Ouvrez le projet starter dans votre IDE

    Localisez le package com.taverne.yagni et familiarisez-vous avec les fichiers présents avant de supprimer quoi que ce soit.

  2. 2

    Implémentez uniquement le besoin validé

    Partez du code de départ, supprimez tout ce qui n'est pas requis, et complétez ce qui manque (la validation du nombre de personnes n'est pas encore implémentée).

  3. 3

    Nettoyez agressivement

    Aucun champ, aucune classe, aucune méthode, aucune interface non utilisée ne doit subsister. Si SonarQube signale du code mort, supprimez-le.

  4. 4

    Justifiez chaque fichier qui reste

    Pour chaque classe dans votre projet final : pouvez-vous dire "cette classe est nécessaire aujourd'hui pour répondre à une exigence validée" ?
